基础排序

1.选择排序
原理： 首先，找到数组中的最小元素，然后将最小元素和第一个元素交换，然后在剩下的元素中找最小的元素，再将该元素和第二个元素交换位置，如此重复，直至数组排序完成，即每次从剩下的元素中选出最小的然后交换位置。因为每一个交换都能确定一个元素的位置，所以交换的次数为N，而比较的次数为，第一次比较为N-1（排除自己和剩下的元素比较），第二次比较N-2….最后一次比较为1，所以比较次数为[（N-1+1）*（N-1)]/2,即≈N^2/2.算法的效率取决于比较的次数

对于长度为N的数组，选择排序需要大约N^2/2次比较和N次交换

2.插入排序
原理：当前索引的左边的元素都是有序的，只是最终的位置是不确定的，所以要将当前索引的元素插入前面的有序元素组中，当前索引为数组的最后索引时，则排序已经完成。插入排序需要的交换操作和数组中倒置的数量相同。

平均情况下插入排序需要~N^2/4次比较以及~N/4次交换。最坏情况下需要~N^2/2次比较和~N^2/2次交换，最好情况下需要N-1次比较和0次交换

3.希尔排序
原理：基于插入排序的快速的排序算法。在插入排序中，对于大规模的乱序数组插入排序很慢，因为它只能通过交换来挪动位置，假设最小的元素在最右端，那么就要通过N-次交换。希尔排序为了加快速度简单地改进了插入排序，交换不相邻的元素以对数组的局部进行排序，并最终用插入排序将局部有序的数组排序。这样可以让在排在靠前的较大元素可以减少交换和比较的次数。关键是选择正确的h序列公式，一般都是取h=长度/2；

在随机无序数组排序中，希尔排序比插入排序的时间快
