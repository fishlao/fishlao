HTTP请求中，accept-encoding: gzip, deflate, sdch, br是指客户端浏览器支持的压缩方式。在HTTP响应中，content-encoding:gzip是指服务端使用了gzip这种压缩方式。缺点就是压缩过程占用CUP资源，客户端浏览器解析也占据了一部分时间.

5.js如何实现异步加载
分清同步加载，异步加载和延迟执行的区别哦，
1)同步加载
<script type="text/javascript" src="http://localhost:3000/javascripts/b.js"></script>
正常没有属性都不加的，就是按照从上到下的情况去执行，下载完之后就会立刻执行；

2）异步加载
方法一：动态创建dom元素；

方法二：使用defer属性
<script defer src="http://localhost:3000/javascripts/b.js"></script>
当js被浏览器异步加载后,不会立刻执行，而是等把html文档解析后才会执行，也就是延迟执行。如果有多个defer属性的js则是按先后顺序执行。这个js会在DOMContentLoaded事件发生之前去执行。

方法三： 使用async（html5）属性
<script async src="http://localhost:3000/javascripts/b.js"></script>
当js被浏览器异步加载后立刻执行，DOMContentLoaded事件并不需要等待js执行完毕

另外，当 HTML 文档解析完成就会触发 DOMContentLoaded，而所有资源加载完成之后，load 事件才会被触发。

6.如何理解js是个单线程

6.1 浏览器的事件循环
js是单线程的理解是js总是执行完一件事情才执行另外一件事情的，即使是有异步函数，定时器什么的，事件监听什么的，会把这些事件放在任务队列里面，而任务队列就是个队列的数据结构，先进先出，不同的异步任务的回调函数会放在不同的任务队列之中。比如一个异步函数，在它请求完毕之后就会把回调函数放在任务队列里面，如果JS主线程把执行栈的东西都执行完了，才会在任务队列中取出任务，比如刚刚说的回调函数，这才会去执行，对于定时器,setTimeout(fn,0)。html5规定了第二个参数的最小值为4毫秒，如果低于这个值，则会自动去增加。

7.同时执行new Promise和settimeout哪个会更快
两个所属于不同的任务队列

其中setTimeout和Promise的任务队列叫做macro-task(宏任务)，当然如我们所想，还有micro-task(微任务)。

macro-task（宏任务）包括：script(整体代码), setTimeout, setInterval, setImmediate, I/O, UI rendering。

micro-task（微任务）包括：process.nextTick, Promises, Object.observe, MutationObserver
总结一下就是先进行一个宏任务，然后所有的微任务后再进行一个宏任务；
