缓存机制规则

我们可以认为浏览器和服务器之间还存在着一个本地的缓存数据库
在客户端第一次访问服务器的时候，此时缓存数据库中没有数据，所以要向服务器请求数据，并将数据缓存起来。

http缓存规则有很多种，
其中强制缓存的时序图如下：

其流程就是浏览器向缓存数据库询问数据有没有失效，如果失效了就向服务器请求最新数据，再将数据存回缓存数据库。

对比缓存时序图

其流程就是浏览器向缓存数据库拿到缓存数据的标识，然后拿这个标识向服务器询问是否已经过期，如果没有过期，就向缓存拿，如果过期了，则返回最新的数据。

强制缓存和对比缓存同时存在的情况下，强制缓存的优先级较高。

强制缓存

主要有两个字段标识Expires/Cache-Control，这两个字段是浏览器在没有缓存数据的情况下，向服务器发起请求，然后服务器在响应体的头部返回。

Expires
这是http1.0的产物，即下次请求时间小于服务器返回的时间，则直接使用缓存，但是服务器生成的时间和浏览器的时间可能会有差异，导致缓存可能会有误差。

Cache-Control
这是http1.1的产物。常见的取值有private、public、no-cache、max-age，no-store，默认为private。
private: 客户端可以缓存
public: 客户端和代理服务器都可缓存（前端的同学，可以认为public和private是一样的）
max-age=xxx: 缓存的内容将在 xxx 秒后失效
no-cache: 需要使用对比缓存来验证缓存数据（后面介绍）
no-store: 所有内容都不会缓存，强制缓存，对比缓存都不会触发

对比缓存

就是浏览器会带着缓存标识去问服务器，这个缓存标志的缓存有没有过期，如果没有过期则返回304标识符，然后浏览器就可以使用缓存了，否则就返回最新的数据



第二次请求的时间比第一次少，因为第二次请求只返回header部分，不返回响应体。

而这种缓存标识主要有两种：
1.Last Modified/If-Modified-Since

Last Modified：浏览器向服务器请求数据的时候，服务器会返回这个字段，告知该资源被修改的最后时间

If-Modified-Since: 当浏览器再次向服务器请求该资源的时候，会带上这个字段，而服务器发现请求中带有这个字段后，会去检查该资源在服务器的最后修改时间是不是大于这个字段，如果大于，则代表该资源被修改过，返回200状态符，并返回响应体，如果小于，则返回304。

2.Etag/If-None-Match(优先级高于1)
Etag:服务器相应请求的时候，告知浏览器该数据的唯一标识符。

If-None-Match:再次请求服务器时，通过此字段通知服务器客户段缓存数据的唯一标识。
服务器收到请求后发现有头If-None-Match 则与被请求资源的唯一标识进行比对，
不同，说明资源又被改动过，则响应整片资源内容，返回状态码200；
相同，说明资源无新修改，则响应HTTP 304，告知浏览器继续使用所保存的cache。
