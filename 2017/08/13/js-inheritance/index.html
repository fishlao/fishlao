2017-08-13
#Js

函数实现继承
构造函数也是函数，只是当他用New操作符去创建对象实例的时候，才拥有了构造函数的功能。函数都有一个prototype对象，默认会有一个constructor属性，指向prototype属性所在的函数对象；而这个对象保存着所有实例共有的方法和属性
1.原型链的继承
原理：A,B实现继承==>通过构造B的实例，然后将B的实例赋值给A的原型对象，这样A的原型对象就有了指向B的原型对象的指针，还可以在A的原型方法中添加额外的属性和方法；A的实例也就拥有了A和B的原型方法了

其实我觉得当使用构造函数去创建对象的时候，会经过以下的步骤
1.创建一个新的对象A
2.将构造函数的执行对象赋值给A
3.调用构造函数
4.返回对象

以前我很好奇为什么构造函数创建出来的实例会有一个[[prototype]]属性，然后指向它的原型对象；其实就是在构造函数创建实例的过程中，就像第一步所说明，会创建一个对象。就像我们平时直接var a = new Object()新建一个对象一样，这个对象的[[prototype]]属性则指向Object的原型对象；而第一步也是这一样的,新建的这个对象实例的[[prototype]]属性自然就指向这个构造函数的原型对象了。

但是这样会出现一些问题

1）本来colors在Super函数构造出来的实例中代表的应该是实例属性，而subType就算继承了它，意义上该属性也应该是实例的属性，即每个对象都有不一样的属性，然后如果通过原型链去构造，此属性就相当于是subType实例的共有属性了，这和JAVA真正意义上的继承是有区别的。和直接写subType.prototype.colors = ["red","green"]没有区别

2）无法向父类的构造方法中传参

2.借用构造函数的继承
构造函数也是一个函数，因此子类其实可以在构造函数中调用父类的方法，通过apply()或者call()，这样就可以解决子类无法向父类传参的问题，而且每个子类都拥有父类属性的副本了。

但是这样也有问题，其实subType构造出来的实例和super的原型方法并没有什么关系，只有在super上声明的属性和方法才能被subType继承，所以subType其实是没有继承到父类的原型方法的，这样函数就无法复用了。因此，也很少单独用这种模型。

3.组合继承

组合继承就是融合了上诉两种继承方法，这样创建出来的对象既可以拥有父类的实例属性，又可以拥有父类的原型属性了；

4.原型式继承
我的天啊！怎么那么多名词…都快要混淆了！！好把，莫慌，其实这个也很简单，因为它也是通过原型链去实现的，只不过比原型链继承更简单一点，这种方法创建的对象不用声明构造函数，而是直接传一个对象作为原型对象，然后返回一个基于该原型对象的对象实例。
好吧，好像是简单了一点点,就是以一个对象作为另一个对象的基础拉拉

这样就省去了创建一个特定的构造函数的代码拉，ECMAScript5也有Object.create()方法的实现，除了可以添加原型对象外，还可以添加额外的对象实例方法，参数格式和Object.defineProperties()的第二个参数形式相同，实现方法

5.寄生式继承
和原型式继承差不多，只不过是可以原型式继承返回的是拿传过来的对象作原型对象，而寄生式继承还可以增强这个对象，例如添加实例方法等,感觉和createObject差不多

6.寄生组合式继承
由于组合原型继承中要调用两次父类的构造函数，而且子类原型中还有些父类不必要的实例属性。所以可以用寄生继承的方法来构造子类的原型，其实子类中要的只不过就是父类的原型对象而已，所以可以先创建一个父类的原型对象副本，再赋值给子类的原型对象。
